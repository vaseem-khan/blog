  1. How to set-up virtualenv

    Sat 03 January 2015 | 1 min read | By Vaseem Ahmed Khan

    On a newly installed Ubuntu system,

    • Install python setuptools

      sudo apt-get update
sudo apt-get upgrade
sudo apt-get install python-setuptools

    • Install pip

      sudo easy_install pip

    • Enable package caching for pip by adding the following in ~/.pip/pip.conf file

      [global]
download-cache = ~/.cache/pip

    • Install virtualenv and virtualenvwrapper

      sudo pip install virtualenv
sudo pip install virtualenvwrapper

    • Add following snippet to ~/.bashrc

      # virtualenvwrapper stuff for .bashrc
VENVW='$(command -v virtualenvwrapper.sh)'
if [ $VENVW != "" ]; then
  . $VENVW
  export WORKON_HOME=$HOME/virtualenvs
  export PROJECT_HOME=$HOME/projects
fi
unset VENVW

    • Read about virtualenv and virtualenvwrapper. Many helper commands like mkvirtualenv, mktmpenv, mkproject, workon, deactivate, cdproject, cdvirtualenv, cdsitepackages, lssitepackages, etc. are provided by virtualenvwrapper.

    • Be creative with various hooks like postactivate, postdeactivate, etc.

  2. Untrack Files From Git

    Fri 26 December 2014 | 1 min read | By Vaseem Ahmed Khan

    When devloping we encounter that a file is tracked under git, you have made changes to it but now needs to push.

    One example can be changes in setings(local changes) file.

    One option is to stash changes. Push and apply stash. Follow these steps at time of every push.

    but I prefer using git update-index

    git update-index --assume-unchanged file_name

    Now this file will be untrack until you want to start keeping track again

    By excuting thois command

    git update-index --no-assume-unchanged file_name

    and we escape from frequent stash before every push.
